# ADR 006: Payment Service Provider (Stripe vs PayPal)

## Status
Pedding

## Context
We are building a gym system that needs subscription payments for memberships and one-time payments for things like day passes, products, and extra services. We want a payment provider that works well with .NET, is free to start, and offers a strong demo/test mode for development.

## Options
Stripe
-Supports subscriptions and one-time payments in one platform
-Has a strong test mode (test cards, easy webhook testing)
-Popular and well-supported in .NET

PayPal
-Has a sandbox and is well-known by users
-But a common .NET checkout SDK on NuGet is deprecated/legacy, which can be risky for a new project

## Decision
In my opinion, Stripe is the best choice because it is easier to use and it has better and more popular .NET SDKs on NuGet.

## Reasoning
Stripe fits our gym needs best because it supports subscriptions and one-time payments very well, and it has the best demo/testing tools for development.

## Consequences
* **Positive:** Faster MVP with Stripe Checkout + webhooks
* **Positive:** Easier membership billing features (trial, failed payment handling, plan changes)
* **Negative:** We must implement webhook handling carefully (retries, duplicate events)
